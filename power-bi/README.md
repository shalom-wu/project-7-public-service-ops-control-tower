# Power BI Package

## ✅ The dashboard ships as a real .pbix

[`public_service_ops_control_tower.pbix`](public_service_ops_control_tower.pbix)
(36MB) — the 4-page dashboard **with the full 1.07M-row extract imported**,
built and saved in Power BI Desktop 2.155. Open it and everything works:
KPI cards, weekly demand trend, SLA/backlog diagnostics, the priority
matrix, and the what-if weight slicers that re-rank hotspots live.

The report was authored as code: the **PBIP project source**
(`*.SemanticModel/model.bim` + `*.Report/report.json`) is generated by
[`../scripts/build_pbip_model.py`](../scripts/build_pbip_model.py) and
[`../scripts/build_pbip_report.py`](../scripts/build_pbip_report.py), opened
in Power BI Desktop, refreshed and saved — so the semantic model, every DAX
measure, and the page layouts are reviewable as text and reproducible.

| File | Purpose |
|---|---|
| `public_service_ops_control_tower.pbix` | **The dashboard** — open in Power BI Desktop (free) |
| `*.SemanticModel/`, `*.Report/`, `*.pbip` | The dashboard's text-based source (PBIP project) |
| [`dashboard_brief.md`](dashboard_brief.md) | What the dashboard is for, page by page, and the headline insight |
| [`data_model.md`](data_model.md) | Star-schema model, relationships, what-if parameters |
| [`dax_measures.md`](dax_measures.md) | Every DAX measure, copy-paste ready |
| [`refresh_instructions.md`](refresh_instructions.md) | How to refresh the data end to end |
| [`manual_build_instructions.md`](manual_build_instructions.md) | Click-by-click steps to rebuild the report from scratch (kept for reference/learning) |
| [`screenshots/`](screenshots) | `pbix_page*.png` = **real Power BI Desktop captures** of the shipped file · `page*.png` = the original design mockups |

Data loads from [`../data/powerbi/`](../data/powerbi) via a `DataFolder`
parameter (Transform data → Edit parameters if your repo lives at a
different path). Regenerate the source tables any time with
`python scripts/run_sql.py`.
